学习资料

视频看完，资料也整理好了。

章节、概览、笔记、练习题和脑图，都从这节课的文稿里生成。你可以直接复习，也可以继续修改、导出，放进自己的学习流程。

章节与概览

把一节课拆成几个自然段落，每段配一句摘要；点一下章节，就回到对应时间点。

笔记与脑图

先生成一版图文笔记，再理成脑图。哪里写得不顺，直接改；课件截图也能插进去。

练习题

按这节课出几道自测题。答错了，回到那一段再看一遍。

长字幕不用每次重读。

Anthropic 走 cache_control，OpenAI-compatible 端把长文稿作为可缓存前缀。反复生成章节、笔记、练习题时，可以命中 prompt caching。

本地搜索，不绑向量库。

文稿搜索用本地关键词匹配。命中项带时间戳，点一下就回到视频里的那一句。

生成后还能改。

笔记、脑图、字幕和课件截图都不是一次性结果。可以改，可以导出，也可以继续整理。

处理管线

导入之后，按步骤处理。

本地视频，或 Bilibili / URL 下载进来后，CoursePilot 会拆成几个阶段跑：抽音频、ASR、字幕纠错、章节、概览、笔记、练习题、脑图。每一步都有进度，某一步失败了，也能单独重试。

01 导入

拖入本地视频，或粘贴 Bilibili / URL 下载。

02 转写

本地 whisper.cpp，或火山、阿里云 DashScope。

03 纠错

按分段 id 修正错别字和口语停顿。

04 生成

章节、概览、笔记、练习题和脑图。

05 复习

搜索文稿，向课程提问，必要时再改笔记。

提问

直接问这节课。

回答只看这节课的字幕，并且回答按句标注出处。课程里讲过，就给你定位到时间点；没讲到，就明确说没有依据。

你问

这一节为什么说学习率太大会发散？

CoursePilot 答

因为每一步更新都会越过损失函数的低点，新的参数反而让误差变大。老师在讲梯度下降动画时指出，步长过大时点会在谷底两侧来回跳，甚至越跳越远。[14:22] 如果课程字幕没有讲到某个细节，我会明确说明没有依据。
